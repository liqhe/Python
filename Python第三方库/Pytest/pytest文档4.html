将pytest与现有的测试套件一起使用¶

Pytest可以与大多数现有的测试套件一起使用，但其行为不同于其他测试运行器，例如nose或Python的默认单元测试框架。

在使用本节之前，您需要安装pytest。

使用pytest运行现有的测试套件¶

假设您想要在某个地方为现有存储库做出贡献。在使用某种版本控制和（可选）设置virtualenv将代码拉入开发空间后，您将需要运行：

cd <repository>
pip install -e .  # Environment dependent alternatives include
                  # 'python setup.py develop' and 'conda develop'

在您的项目根目录中。这将为站点包中的代码设置一个符号链接，允许您在测试运行时编辑代码，就像安装它一样。

在开发模式下设置项目可以避免每次要运行测试时重新安装，并且比使用sys.path将测试指向本地代码更简单。

还考虑使用tox。
