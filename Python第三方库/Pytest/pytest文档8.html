Monkeypatching / mocking模块和环境¶

有时，测试需要调用依赖于全局设置的功能，或调用无法轻松测试的代码（如网络访问）。该monkeypatch夹具可以帮助您安全地设置/删除属性，字典项或环境变量或修改sys.path导入。请参阅monkeypatch博客文章，了解一些介绍材料并讨论其动机。

简单的例子：monkeypatching函数¶

如果您想假装os.expanduser返回某个目录，可以monkeypatch.setattr()在调用使用它的函数之前使用该方法修补此函数：

# content of test_module.py
import os.path
def getssh(): # pseudo application code
    return os.path.join(os.path.expanduser("~admin"), '.ssh')

def test_mytest(monkeypatch):
    def mockreturn(path):
        return '/abc'
    monkeypatch.setattr(os.path, 'expanduser', mockreturn)
    x = getssh()
    assert x == '/abc/.ssh'

这里我们的测试函数monkeypatches os.path.expanduser然后调用一个调用它的函数。测试功能完成后，os.path.expanduser修改将被撤消。

示例：阻止远程操作的“请求” ¶

如果要阻止“请求”库在所有测试中执行http请求，您可以执行以下操作：

# content of conftest.py
import pytest
@pytest.fixture(autouse=True)
def no_requests(monkeypatch):
    monkeypatch.delattr("requests.sessions.Session.request")

将为每个测试函数执行此autouse fixture，它将删除该方法，request.session.Session.request 以便在测试中创建http请求的任何尝试都将失败。

注意

注意，不建议修补的内置功能，如open， compile等，因为它可能会破坏pytest的内部。如果这是不可避免的，通过--tb=native，--assert=plain并--capture=no可能有所帮助，虽然没有保证。

注意

请注意，stdlibpytest使用的修补函数和某些第三方库本身可能会破坏pytest，因此在这些情况下，建议使用它 MonkeyPatch.context()来修补要测试的块：

import functools


def test_partial(monkeypatch):
    with monkeypatch.context() as m:
        m.setattr(functools, "partial", 3)
        assert functools.partial == 3

有关详细信息，请参阅问题＃3290。
