临时目录和文件¶

该tmp_path夹具¶

3.9版中的新功能。

您可以使用tmp_pathfixture，它将提供在基本临时目录中创建的测试调用唯一的临时目录。

tmp_path是一个pathlib/pathlib2.Path对象。以下是测试用法的示例：

# content of test_tmp_path.py
import os

CONTENT = u"content"


def test_create_file(tmp_path):
    d = tmp_path / "sub"
    d.mkdir()
    p = d / "hello.txt"
    p.write_text(CONTENT)
    assert p.read_text() == CONTENT
    assert len(list(tmp_path.iterdir())) == 1
    assert 0

该tmp_path_factory夹具¶

3.9版中的新功能。

这tmp_path_factory是一个会话范围的夹具，可用于从任何其他夹具或测试创建任意临时目录。

它旨在替换tmpdir_factory和返回pathlib.Path实例。

有关详细信息，请参阅tmp_path_factory API。

'tmpdir'灯具¶

您可以使用tmpdirfixture，它将提供在基本临时目录中创建的测试调用唯一的临时目录。

tmpdir是一个py.path.local对象，提供os.path方法等。以下是测试用法的示例：

# content of test_tmpdir.py
import os
def test_create_file(tmpdir):
    p = tmpdir.mkdir("sub").join("hello.txt")
    p.write("content")
    assert p.read() == "content"
    assert len(tmpdir.listdir()) == 1
    assert 0

运行它会导致传递测试，除了我们用来查看值的最后 一行：assert 0

'tmpdir_factory'夹具¶

版本2.8中的新功能。

这tmpdir_factory是一个会话范围的夹具，可用于从任何其他夹具或测试创建任意临时目录。

例如，假设您的测试套件需要磁盘上的大图像，这是在程序上生成的。tmpdir您可以为每个会话生成一次以节省时间，而不是为每个使用它的测试计算相同的图像：

# contents of conftest.py
import pytest


@pytest.fixture(scope="session")
def image_file(tmpdir_factory):
    img = compute_expensive_image()
    fn = tmpdir_factory.mktemp("data").join("img.png")
    img.save(str(fn))
    return fn


# contents of test_image.py
def test_histogram(image_file):
    img = load_image(image_file)
    # compute and test histogram

有关详细信息，请参阅tmpdir_factory API。

默认的基本临时目录¶

默认情况下，临时目录创建为系统临时目录的子目录。基本名称将是pytest-NUM其中 NUM将与每个测试运行递增。此外，将删除超过3个临时目录的条目。

您可以覆盖默认的临时目录设置，如下所示：

pytest --basetemp=mydir

在本地计算机上分发测试时，请pytest注意为子进程配置basetemp目录，以便所有临时数据都落在单个每个测试运行的basetemp目录下。
