捕获stdout / stderr输出¶

默认的stdout / stderr / stdin捕获行为¶

在测试执行期间，发送到stdout和stderr捕获的任何输出。如果测试或设置方法失败，则通常会显示相应捕获的输出以及故障回溯。（可以通过--show-capture命令行选项配置此行为）。

此外，stdin设置为“null”对象，在尝试从中读取时将失败，因为在运行自动化测试时很少需要等待交互式输入。

默认情况下，通过拦截对低级文件描述符的写入来完成捕获。这允许捕获简单打印语句的输出以及测试启动的子进程的输出。

设置捕获方法或禁用捕获¶

有两种方法pytest可以执行捕获：

  • 文件描述符（FD）级别捕获（默认）：将捕获进入操作系统文件描述符1和2的所有写入。
  • sys级别捕获：只写Python文件sys.stdout 和sys.stderr将被捕获。不执行对文件描述符的写入捕获。

您可以从命令行影响输出捕获机制：

pytest -s            # disable all capturing
pytest --capture=sys # replace sys.stdout/stderr with in-mem files
pytest --capture=fd  # also point filedescriptors 1 and 2 to temp file

使用print语句进行调试¶

默认捕获stdout / stderr输出的一个主要好处是可以使用print语句进行调试：

# content of test_module.py

def setup_function(function):
    print("setting up %s" % function)

def test_func1():
    assert True

def test_func2():
    assert False

从测试函数访问捕获的输出¶

的capsys，capsysbinary，capfd，和capfdbinary固定装置允许访问测试执行期间创建的stdout / stderr输出。下面是一个示例测试函数，它执行一些与输出相关的检查：

def test_myoutput(capsys):  # or use "capfd" for fd-level
    print("hello")
    sys.stderr.write("world\n")
    captured = capsys.readouterr()
    assert captured.out == "hello\n"
    assert captured.err == "world\n"
    print("next")
    captured = capsys.readouterr()
    assert captured.out == "next\n"

该readouterr()调用到目前为止对输出进行快照 - 并且将继续捕获。测试功能完成后，将恢复原始流。使用capsys这种方式可以使您的测试免于设置/重置输出流，并且还可以与pytest自己的每次测试捕获进行良好的交互。

如果要在filedescriptor级别捕获，可以使用capfd提供完全相同接口的fixture，但也允许捕获直接写入操作系统级输出流（FD1和FD2）的库或子进程的输出。

版本3.3中的新功能。

返回值从readouterr更改为namedtuple具有两个属性的a，out和err。

版本3.3中的新功能。

如果测试中的代码写入非文本数据，您可以使用从该方法capsysbinary返回的fixture 来捕获它。该夹具目前仅在python 3中可用。bytesreadouterrcapfsysbinary

版本3.3中的新功能。

如果测试中的代码写入非文本数据，您可以使用从该方法capfdbinary返回的fixture 来捕获它。该夹具上的文件描述符级别运行。bytesreadouterrcapfdbinary

3.0版中的新功能。

到测试内暂时禁用捕获，两者capsys 并capfd有一个disabled()可被用作上下文管理方法，禁用内部捕获with块：

def test_disabling_capturing(capsys):
    print("this output is captured")
    with capsys.disabled():
        print("output not captured, going directly to sys.stdout")
    print("this output is also captured")
